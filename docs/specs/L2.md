# L2 Requirements

## L1-1: Enhanced Draw.io Skill Instructions

### L2-1.1: Expanded Diagram Type Coverage
The skill instructions shall cover all major diagram categories from the draw.io examples gallery: Flowcharts, UML (all 8 subtypes), C4 Model (4 levels), Network/Infrastructure, AWS Architecture, Swimlane/Cross-Functional, BPMN, Sequence, ER, Mind Maps, Org Charts, Gitflow, Kanban, Rack/Cabinet, Mockups/Wireframes, Timelines/Roadmaps, and Infographics.

**Acceptance Criteria:**
- Each diagram type has a dedicated section with specific shape styles, layout rules, and a complete XML template example
- Templates use correct mxCell syntax with proper parent-child relationships and style strings

### L2-1.2: Professional Styling and Theming System
The skill shall define a comprehensive theming system with multiple professional color palettes, typography rules, shadow/gradient options, and consistent styling patterns derived from the best draw.io examples.

**Acceptance Criteria:**
- At least 3 named color themes (e.g., Corporate Blue, Modern Dark, Clean Light) with specific hex values
- Font hierarchy rules (title, heading, body, annotation sizes and families)
- Shape styling includes fill, stroke, shadow, rounded corners, and opacity guidelines

### L2-1.3: Layout and Composition Rules
The skill shall provide detailed spatial composition rules including grid alignment, spacing formulas, flow direction conventions, grouping/containment patterns, legend placement, and multi-page diagram strategies.

**Acceptance Criteria:**
- Specific pixel values for margins, padding, gaps between elements
- Rules for when to use containers, swimlanes, groups, and boundaries
- Guidelines for legend/key placement and content
- Multi-page linking instructions with proper XML syntax

### L2-1.4: Connector and Routing Best Practices
The skill shall provide comprehensive connector guidance including routing styles, waypoint usage, line jumps, label placement, arrowhead types, and relationship notation (UML, ER crow's foot, etc.).

**Acceptance Criteria:**
- Style strings for each connector type (orthogonal, curved, straight, entity-relation)
- Waypoint shape usage instructions for clean junctions
- Line jump configuration for crossing connections
- Arrowhead catalog with style strings for each type

### L2-1.5: Complete XML Template Library
The skill shall include complete, valid, ready-to-use XML templates for the top 10 most common diagram types that can be saved directly as .drawio files.

**Acceptance Criteria:**
- Each template is a complete, valid mxGraphModel XML document
- Templates include proper id sequencing, parent references, geometry, and styles
- Templates demonstrate the styling and layout best practices from other L2 requirements
- Templates cover: Flowchart, Sequence Diagram, Class Diagram, ER Diagram, Network Diagram, C4 Context, Swimlane, Architecture, Org Chart, and Mind Map

### L2-1.6: Anti-Patterns and Common Mistakes
The skill shall document common diagram mistakes to avoid, derived from contrasting good examples with typical poor practices.

**Acceptance Criteria:**
- At least 8 documented anti-patterns with explanations of why they're problematic
- Each anti-pattern has a corresponding "do this instead" recommendation

## L1-2: Iterative Diagram Generation and Quality Validation

### L2-2.1: Diagram Generation Test Harness
A test script shall generate diagrams using the skill instructions, save them to a temp folder, and evaluate quality against reference criteria.

**Acceptance Criteria:**
- Script generates .drawio XML files in a temp directory
- Script can be run repeatedly (40 iterations) to test consistency
- Each generated diagram is valid XML that opens in draw.io
- Quality metrics are logged for each iteration

### L2-2.2: Quality Scoring and Variance Measurement
Each generated diagram shall be scored against a quality rubric covering structure, styling, labeling, layout, and completeness, with variance tracked across iterations.

**Acceptance Criteria:**
- Quality rubric covers: valid XML structure, proper id/parent hierarchy, consistent styling, appropriate spacing, meaningful labels, correct flow direction, legend presence, color palette adherence
- Variance is calculated and reported across iterations
- Final variance after 40 iterations is documented
